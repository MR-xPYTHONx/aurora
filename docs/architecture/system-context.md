# System Context

## Users
Users interact with the Aurora platform through the web application to access features, manage content, and receive AI-powered assistance.

## Web App
The web app is the primary interface for users. It handles client-side interactions, sends requests to backend services, and presents results in a responsive experience.

## Backend
The backend coordinates application logic, business rules, and service-to-service communication. It acts as the main orchestration layer for the platform.

## AI Service
The AI service processes requests that require intelligence or generation. It integrates model capabilities to support features such as chat, reasoning, and content assistance.

## Database
The database stores application data, user-related information, and system state needed for persistence and retrieval.

## External AI Providers
External AI providers supply the underlying model capabilities used by the AI service. These providers are integrated through APIs and are abstracted behind the platform’s service layer.
